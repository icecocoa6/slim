49 // This small disassembler is very limited
50 // in its functionality, and in fact does only the bare minimum required by the
51 // preamble patching utility. It may be useful for other purposes, however.
52 //
53 // The limitations include at least the following:
54 // -# No support for coprocessor opcodes, MMX, etc.
55 // -# No machine-readable identification of opcodes or decoding of
56 // assembly parameters. The name of the opcode (as a string) is given,
57 // however, to aid debugging.
58 //
59 // You may ask what this little disassembler actually does, then? The answer is
60 // that it does the following, which is exactly what the patching utility needs:
61 // -# Indicates if opcode is a jump (any kind) or a return (any kind)
62 // because this is important for the patching utility to determine if
63 // a function is too short or there are jumps too early in it for it
64 // to be preamble patched.
65 // -# The opcode length is always calculated, so that the patching utility
66 // can figure out where the next instruction starts, and whether it
67 // already has enough instructions to replace with the absolute jump
68 // to the patching code.
69 //
70 // The usage is quite simple; just create a MiniDisassembler and use its
71 // Disassemble() method.
72 //
73 // If you would like to extend this disassembler, please refer to the
74 // IA-32 Intel® Architecture Software Developer’s Manual Volume 2:
75 // Instruction Set Reference for information about operand decoding
76 // etc.
77 class MiniDisassembler {

84  // @param address_default_32_bits If true, the default address size is

92  // Attempts to disassemble a single instruction starting from the
93  // address in memory it is pointed to.
94  //
95  // @param start Address where disassembly should start.
96  // @param instruction_bytes Variable that will be <b>incremented</b> by
97  // the length in bytes of the instruction.
98  // @return enItJump, enItReturn or enItGeneric on success. enItUnknown
99  // if unable to disassemble, enItUnused if this seems to be an unused
100  // opcode. In the last two (error) cases, cbInstruction will be set
101  // to 0xffffffff.
102  //
103  // @post This instance of the disassembler is ready to be used again,
104  // with unchanged defaults from creation time.
105  InstructionType Disassemble(unsigned char* start, unsigned int& instruction_bytes);
